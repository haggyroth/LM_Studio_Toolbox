A feature-rich [LM Studio](https://lmstudio.ai/) plugin that gives your local LLM a full suite of tools for autonomous coding, research, file management, and more. It acts as a bridge between the model and your local environment.

> **This is an independent fork** of [Beledarian's LM Studio Toolbox](https://github.com/Beledarian/Beledarians_LM_Studio_Toolbox), maintained at [haggyroth/LM_Studio_Toolbox](https://github.com/haggyroth/LM_Studio_Toolbox).

## Key Features

### File System Mastery

- **Full Control:** Create, read, update, delete, move, and copy files and directories.
- **Safe & Secure:** All operations are sandboxed to your workspace; path traversal is blocked. Configurable `protectedPaths` prevents the model from touching sensitive directories.
- **Smart Editing:** `replace_text_in_file`, `multi_replace_text`, `insert_at_line`, `delete_lines_in_file`, and `edit_file_with_diff` for surgical edits without rewriting entire files.
- **Powerful Search:** `search_directory` provides grep-style multi-file search with context lines, regex support, exclude patterns, and a configurable `max_matches` cap.
- **File Watching:** `watch_file` / `watch_directory` for reactive workflows that respond to changes on disk.

### Autonomous Sub-Agents

- **Role-Based Delegation:** `consult_secondary_agent` dispatches to a specialised sub-agent — `coder`, `reviewer`, `researcher`, `debugger`, `tester`, `documenter`, `planner`, `data_analyst`, or `general`.
- **Role Chaining:** Pass a `chain` list to run multiple roles in sequence, each receiving the previous output (e.g., `coder → tester → reviewer`).
- **Auto-Save:** Code generated by sub-agents is automatically detected and saved to disk.
- **Execution Log:** Every sub-agent response includes a compact `[Execution log]` of tool calls and outcomes for transparency.
- **Token Tracking:** Sub-agent footers report prompt + completion token counts and elapsed time.
- **429 Retry:** Built-in rate-limit retry with `Retry-After` header support and exponential backoff for transient network errors.
- **Mid-Task Steering:** `interrupt_sub_agent` injects a correction message into a running sub-agent loop without cancelling it.
- **Result Replay:** `get_sub_agent_result` re-reads the last completed result without launching a new session.

### Persistent Memory

- **CRUD tools:** `save_memory`, `list_memories`, `search_memories`, `update_memory`, `delete_memory`.
- **Auto-injection:** Top 50 memories are injected into context at the start of every conversation — no explicit lookup needed.
- **Deduplication:** Exact-match (case-insensitive) dedup prevents duplicate facts from accumulating.
- **Fallback backend:** Pure-JS JSON backend activates automatically if the native SQLite addon can't load (e.g. inside LM Studio's sandboxed runtime), so memory always works.
- **Auto-capture:** Optional `memoryAutoCapture` mode distills key facts from each session automatically.

### Code Execution

- **Sandboxed Python:** `run_python` uses `sys.addaudithook` to block network access, subprocess spawning, and writes outside the workspace.
- **JavaScript:** `run_javascript` runs Node.js code in a temp file.
- **Test Runner:** `run_test_command` executes your project's test suite (`npm test`, `pytest`, etc.) from inside a sub-agent loop.
- **Linting:** `lint_project` runs your project's linter and returns structured output.
- **Terminal:** `execute_command` (background) and `run_in_terminal` (interactive) for shell access.

> [!WARNING]
> Enabling shell or terminal execution allows the model to run arbitrary commands on your system.

### Code Intelligence (TypeScript / JavaScript)

- **`find_symbol`:** Locate where a function, class, or variable is defined using AST analysis — no grep false positives.
- **`find_usages`:** Find all call sites for a symbol across the workspace.
- **`rename_symbol`:** Atomically rename an identifier across the entire workspace in a single operation.

### Web & RAG

- **Search:** `web_search` (DuckDuckGo + HTML fallback), `wikipedia_search`.
- **Fetch:** `fetch_web_content`, `rag_web_content` (with SSRF protection — private IPs blocked, redirects re-validated).
- **Browser Automation:** Persistent `browser_session_open` / `browser_session_control` / `browser_session_close` flow for multi-step navigation, form filling, and JavaScript evaluation.
- **Screenshot:** `capture_screenshot` — open a URL, capture a screenshot, and save it to your workspace in one call.
- **Local RAG:** `rag_local_files` — semantic search over workspace files with embedding cache.

### Git & GitHub

- **Git:** `git_status`, `git_diff` (word-level mode), `git_log`, `git_add`, `git_commit`, `git_pull`, `git_push`, `git_checkout`.
- **GitHub CLI:** `gh_create_issue`, `gh_list_issues`, `gh_create_pr`, `gh_list_prs`, `gh_view_pr_diff`, and more.

### Data & Workspace

- **`query_database`:** Read-only SQL queries against a SQLite file in your workspace.
- **`query_csv`:** Query a CSV file with SQL-style filtering and sorting.
- **`transform_json`:** Transform a JSON file using a JavaScript expression.
- **`analyze_project`:** Scan the workspace and produce a structured project summary.
- **`tool_usage_stats`:** Audit log of tool call counts and patterns for the current session.
- **`save_session_note`:** Persist a note about the current session for future reference.
- **Workspace Profiles:** `save_workspace_profile`, `switch_workspace_profile`, `list_workspace_profiles` for quickly switching between project contexts.

### Custom Plugins

Drop a `.js` file into `~/.lm-studio-toolbox/plugins/` to register your own tools at startup — no forking required. See [PLUGINS.md](PLUGINS.md) for the format.

## Installation

```bash
git clone https://github.com/haggyroth/LM_Studio_Toolbox.git
cd LM_Studio_Toolbox
npm install
lms push   # install into LM Studio permanently
# or
lms dev    # development mode with hot reload
```

### Development

```bash
git clone https://github.com/haggyroth/LM_Studio_Toolbox.git
cd LM_Studio_Toolbox
npm install
lms dev    # hot-reloads on every source change
```

Run the full quality pipeline before submitting a PR:

```bash
npm run ci   # typecheck → lint → build → test
```

## Configuration

All settings are accessible in the LM Studio **Plugins** tab. Key options:

| Setting | Description | Default |
|---------|-------------|---------|
| Enable Secondary Agent | Enables the `consult_secondary_agent` tool | off |
| Secondary Agent Endpoint | OpenAI-compatible endpoint for sub-agents | `http://localhost:1234` |
| Secondary Model ID | Model to use for sub-agents | (blank = same as main) |
| Sub-Agent Loop Limit | Max LLM turns per sub-agent run | 8 |
| Sub-Agent Time Limit | Wall-clock deadline in seconds | 600 |
| Sub-Agent: Show Execution Log | Append a compact tool-call log to each response | on |
| Sub-Agent Profiles | JSON map of role → system prompt overrides | (built-in defaults) |
| Enable Memory | Activates persistent memory tools | off |
| Memory Auto-Capture | Auto-distill key facts from each session | off |
| Default Workspace Path | Startup working directory | (LM Studio home) |
| Allow Code Execution | Enables Python, JS, shell, terminal tools | off |
| Allow Browser Control | Enables browser automation tools | off |
| Allow GitHub Tools | Enables `gh_*` CLI tools | off |
| Protected Paths | Comma-separated paths the model cannot access | (empty) |
